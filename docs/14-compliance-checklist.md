# Section 14: Pre-publish compliance checklist

Run this before any asset ships. If any answer is no, fix it before publishing.

- [ ] Person-first throughout, and no use of _impaired_ in any form or the acronym _BVI_?
- [ ] No em dashes anywhere?
- [ ] Correct context identified, and the right voice applied (commercial, government, mission, donor, or internal)?
- [ ] For commercial or government work, does capability lead and mission stay a closer, if it appears at all?
- [ ] For mission work, does the person's agency lead, with Envision enabling rather than rescuing?
- [ ] Envision leads as the primary brand, with programs as plain descriptors except the four named lockups?
- [ ] No invented names or lockups, and naming deferred to the Standard Naming Units document?
- [ ] Location language correct: no _headquartered in Wichita_, Farmers Branch used for facility accuracy?
- [ ] Foundations written as Envision's Dallas Foundation and Envision's Wichita Foundation, with the apostrophe?
- [ ] Donor names used only in donor-facing contexts?
- [ ] Claims measurable and verified, with no invented certifications or metrics?
- [ ] No forbidden patterns from [Section 6](06-forbidden-patterns.md), and no AI-sounding or decorative phrasing?
- [ ] Sentence case, active voice, short paragraphs, and one clear call to action where relevant?
- [ ] For designed or visual assets: approved logo, palette, and typefaces used, with the logo unaltered and no retired marks, per [Section 12](12-visual-and-design-direction.md)?
- [ ] For designed or visual assets: WCAG 2.1 AA contrast met, alt text on all images, and meaning never carried by color alone?
